#学习使用git版本控制工具#
现在我们常见的版本控制工具有两种：SVN和Git，SVN是一种集中式版本控制工具，就是当多个人共同开发一个项目是，用来共享资源用的，这种工具的保密性强，权限的要求严格，适合企业等使用。Git是一个开源的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理。分布式相比于集中式的最大区别在于开发者可以提交到本地，每个开发者通过克隆（git clone），在本地机器上拷贝一个完整的Git仓库。但它没有权限的设置，谁都可以下载，安全性和保密性都有所欠缺。
使用Git之前，我们首先要在Github官网注册一个Github账号，然后安装Git工具，最后就是敲命令了。以下是我使用Git的过程中使用到的一些命令，应该也是最常用的命令吧，初学者照着我的顺序来就可以成功将自己的项目分享到Github上了。

  1. 设置用户信息（自我感觉这个没什么用）
    $ git config –global user.name “Jack”
    $ git config –global user.email “12345676@163.com”
  2. 设置完后可以查看一下,检查自己的设置是否成功
    $ git config user.name
    $ git config user.email
  3. 在当前目录下新建一个代码库
    $ git init
  4. 添加指定文件到暂存区
    $ git add [file1] [file2] …
    或者$ git add . 可以将当前目录下的所有文件都添加到暂存区
  5. 将文件从暂存区提交到本地仓库
    $ git commit -m “这里写提示信息”
  6. 添加远程版本库链接
    $ git remote add origin “这里写您的github网址”
  7. 提交文件到github库
    $ git push -u origin master
    运行完这句话回车后会弹出一个对话框，把您github的用户名和密码正确输入
  8. 显示所有的远程仓库
    $ git remote -v
    然后在浏览器中输入https://用户名。github.io就可以看到您刚才共享的项目了
    将项目下载到本地有以下几个步骤：
    本地在某个目录下新建一个文件夹用来做项目仓库然后在该文件夹内打开git bash
    $ git clone https://.......git
    等待克隆完成以后就可以在本地的文件夹中看到我们下载的项目了。
    大家都知道，Git不仅能上传项目还可以发布博客，这里我们通常使用Hexo配合使用，
    首先，Hexo是NodeJS的一个插件，我们需要先安装nodeJS,然后再用npm(nodeJS的包管理器)来下载需要的插件。下面是我整理的操作步骤：
  9. $ npm install hexo-cli -g
  10. $ hexo init
  11. 在D盘新建一个文件夹，用来保存你的博客，然后$ npm install
  12. 生成静态文件 $ hexo g
  13. 开启浏览器 $ hexo s,然后在127.0.0.1：4000查看本地文件
  14. 然后修改_config.yml文件
    deploy:
    type: git
    repository: github网址
    branch: master
  15. ctrl+c停止服务器
  16. 以上步骤中如果出现错误就执行$ npm install hexo-deploy-git –save
  17. $ hexo d 将项目发布到网上
  18. 如果要新建项目 $ hexo new “文件名”
  19. 然后再$ hexo d
